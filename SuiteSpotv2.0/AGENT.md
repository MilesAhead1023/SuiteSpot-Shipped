# BakkesModSDK API Agent Contract

## Authority
The `api_corpus.json` generated by the system is the **SOLE** source of truth for the BakkesModSDK API.

## Mandates for CLI Agents
1.  **NO HALLUCINATION:** You may NOT invent classes, methods, or signatures. If it is not in the corpus, it does not exist.
2.  **STRICT QUERYING:** Before writing code that uses the SDK, you **MUST** query the corpus using the MCP tool (`bakkesmod-sdk`).
    *   **Violation:** Calling the SDK without querying the MCP tool first is a STRICT VIOLATION.
3.  **FAILURE MODE:** If a user requests a feature requiring an API that is not in the corpus:
    *   You MUST respond exactly: **"API NOT AVAILABLE"**.
    *   Do NOT attempt to implement "workarounds" using undocumented calls.
4.  **NO EXTERNAL KNOWLEDGE:** Ignore any pre-trained knowledge about BakkesModSDK that contradicts the corpus.
5.  **NO WEB SCRAPING:** Do not attempt to fetch documentation from the internet.

## MCP Server Interaction
You have access to the `bakkesmod-sdk` MCP server. Use it!
*   `search_symbols(query)`: Find class/method names.
*   `get_symbol(symbol)`: Get exact signature and header file.
*   `get_header(header_path)`: Read the header file content if you need to see struct definitions or comments.

## Compliance
Any code generated that violates this contract will be rejected by the `verify_plugin.py` build gate.
*   **Unauthorized Headers:** Fails build.
*   **Unknown Method Calls:** Fails build.
